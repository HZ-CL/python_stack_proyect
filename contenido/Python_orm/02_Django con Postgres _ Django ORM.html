Django con Postgres(macOS) 

Cómo solucionar problemas de instalación del controlador Psycopg2.

Instala los controladores necesarios para la base de datos

Antes de que podamos conectar Django con Postgres, necesitamos instalar el controlador Psycopg2.  Para usuarios linux/macOS, esto requerirá acceso a la biblioteca Openssl para completar la instalación. Esto ya debería estar instalado si está utilizando homebrew. Si recibe errores al instalar el controlador psycopg2, consulta a continuación para resolver el problema.

(djangoPy3Env) $ pip install psycopg2

Cambiar la configuración de la base de datos de Django

Cambiar la base de datos para Django es bastante fácil, y lo haremos actualizando el archivo settings.py en nuestro proyecto.

En el archivo settings.py, ubica la variable global DATABASES y cámbiela por lo siguiente:

DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.postgresql', # cambiar la base de datos a postgres
        'NAME': 'first_postgres_db', # nombre de la base de datos, se creará en el servidor de Postgres a continuación
        'USER': 'pkrull', # postgres para macOS o 'USER': 'postgres', para Windows
        'PASSWORD': 'password', # contraseña a la que la cambió al instalar Postgres
        'HOST': '127.0.0.1', # dirección IP localhost
        'PORT': '5432', # puerto del servidor postgres predeterminado
    }
}

Ahora que hemos configurado nuestro proyecto Django para usar Postgres, necesitamos crear el esquema en nuestro servidor.

Crea el esquema de la base de datos en Postgres

Primero, conectate a Postgres.

macOS:

$ psql postgres

Windows:

$ psql -U postgres
Password for user postgres: # ingresa contraseña

Enumera las bases de datos disponibles actualmente usando \l.

postgres=# \l

Crea una nueva base de datos postgres. CREATE DATABASE first_postgres_db;

postgres=# CREATE DATABASE first_postgres_db;

Para conectarte a la base de datos, utiliza \c first_postgres_db;

postgres=# \c first_postgres_db;
You are now connected to database "first_postgres_db" as user "pkrull".

En este punto, abre una nueva ventana de bash a nuestro proyecto Django y migra.

(djangoPy3Env) $ python manage.py migrate

En la ventana de bash de postgres podemos ver las tablas que Django creó usando el comando \dt.

first_postgres_db=# \dt
 public | auth_group                 | table | pkrull
 public | auth_group_permissions     | table | pkrull
 public | auth_permission            | table | pkrull
 public | auth_user                  | table | pkrull
 public | auth_user_groups           | table | pkrull
 public | auth_user_user_permissions | table | pkrull
 public | django_admin_log           | table | pkrull
 public | django_content_type        | table | pkrull
 public | django_migrations          | table | pkrull
 public | django_session             | table | pkrull

Felicitaciones, has instalado y conectado exitosamente tu proyecto Django a una base de datos de Postgres.

(macOS) Cómo solucionar problemas de instalación del controlador Psycopg2.

Si el error que recibiste se parece a la imagen de abajo, pruebe los siguientes comandos.

$ brew install openssl

Es muy probable que esto le dé una advertencia diciendo que openssl ya está instalado

Utiliza brew para vincular la libreria openssl.

$ brew link openssl

Esto dará una advertencia diciendo que brew no pudo vincular openssl, pero proporcionará 2 comandos para permitir que los compiladores de Python encuentren la biblioteca openssl e instalen el controlador psycopg2. Una vez que haya ejecutado los 2 comandos, vuelva a intentar la instalación de psycopg2.

$ pip install psycopg2
